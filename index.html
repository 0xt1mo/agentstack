The OpenStack Moment for AI Agents

In 2010, enterprises faced a familiar problem. Cloud computing was transforming infrastructure, but every workload ran on a single vendor's platform — locked in, opaque, impossible to audit. OpenStack solved it by building the infrastructure layer underneath the cloud.

The AI agent ecosystem is at the same inflection point. Agents are deployed across every enterprise function. The productivity gains are real. But every agent runs on a framework built for experimentation, not production — with no shared security model, no audit trail, and no governance layer.

AgentStack is the infrastructure layer underneath every agent framework.

The Problem: Trust

Enterprise AI agent deployment is not blocked by capability. The models are capable. The use cases are clear. It is blocked by trust.

Every framework handles security at the wrong layer — inside the model, via prompts and guardrails that rely on the LLM to police itself. A sufficiently crafted prompt injection defeats it. An indirect injection via a malicious document defeats it.

The result is a security model built on the assumption that the thing being secured will cooperate in its own security. No enterprise security architect would accept this in any other context.

AstridOS — The Agent Runtime

A user-space microkernel. Agents don't run alongside it — they run on top of it. Every framework, LLM frontend, and tool is a user-space process communicating over IPC. The enforcement layer lives in the kernel — below everything.

You cannot prompt-inject your way past a kernel boundary.

  • WASM Capsule Sandbox — every tool runs in architectural isolation
  • Ed25519 Authorization — actions signed cryptographically, not by prompt
  • Immutable Audit — chain-linked, tamper-evident records
  • Human-in-the-Loop — approval gates as kernel primitives
  • Budget Enforcement — compute and API limits at runtime level
SIF — The Agent Firewall

Every message — user↔agent, agent↔agent, agent↔tool — passes through SIF before it reaches the model and before it leaves the stack. Sub-20ms classification latency.

Policy Dashboard

Content-based semantic security policies applied uniformly across every agent, framework, and session. Configure without model retraining or framework modification. Rules update dynamically and propagate immediately.

  • Prompt injection & indirect injection detection
  • Data leakage & IP exfiltration prevention
  • PII and credentials handling rules
  • Domain-specific ontologies (finance, healthcare, legal)
  • Transaction approval thresholds
  • Competitor & confidential information controls
  • Custom classification models via ONNX
  • No system restart for rule changes
  • Full GUI management (enterprise)
